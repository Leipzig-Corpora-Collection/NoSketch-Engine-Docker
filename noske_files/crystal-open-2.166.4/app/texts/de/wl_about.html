Mithilfe der Wortliste können Sie verschiedene Häufigkeitslisten erstellen:
  • Nomen, Verben, Adjektive und andere Wortarten
  • Wörter, die mit bestimmten Buchstaben beginnen, enden oder diese enthalten
  • %[wordForm|Wortformen]%, %[tag|Tags]%, %[lemma|Lemmata]% und andere %[attribute|Attribute]%
oder eine Kombination dieser drei Optionen.
Die Häufigkeit kann in der Wortliste in drei verschiedenen Varianten angezeigt werden: %[frequency|Häufigkeit]%, %[relfreq|Häufigkeit pro Million]% und ARF.
  • Wie funktioniert die Wortliste?
    Die Wortliste arbeitet auf der Ebene von %[token|Token]%. Standardmäßig werden nur Wörter angezeigt, weil Nichtwörter automatisch aussortiert werden.

    Die Wortliste kann in den Einstellungen nach minimaler und maximaler Häufigkeit eingegrenzt werden.

    Häufigkeitslisten können für eine Auswahl konkreter Wörter erstellt werden. Wählen Sie die Option aus dieser Liste: in der erweiterten Suche und geben sie die Wörter ein, deren Häufigkeit im gewählten Korpus berechnet werden soll.

    %[regex|Reguläre Ausdrücke (Regex)]% können genutzt werden, um komplexe Kriterien für die Wörter zu definieren, die in die Häufigkeitsliste aufgenommen werden sollen.
  • Mehr Informationen in den Ergebnissen anzeigen
    Mit der Option Darstellen als können Sie dem Ergebnis mehr Attribute hinzufügen. Das kann beispielsweise eine Häufigkeitsliste von Wortformen sein, in der neben den Wortformen zusätzlich die Lemmata angezeigt werden. Sie können bis zu 3 Attribute auswählen.

    Einschränkungen
    Wenn die Option Darstellen als ausgewählt ist, wird zunächst eine Konkordanz mit allen den Kriterien entsprechenden %[token|Token]% erzeugt. Die Häufigkeit wird dann aus dieser Konkordanz berechnet. Diese Konkordanz ist aus technischen Gründen auf eine Größe von 10 Millionen Konkordanzzeilen begrenzt. Das kann bei sehr großen Korpora dazu führen, dass die Häufigkeitsberechnung nicht auf dem gesamten Korpus basiert.

    Diese Einschränkung kann nicht aufgehoben werden. Sketch Engine kann jedoch in solchen Fällen gegen eine Gebühr Häufigkeitslisten aus dem gesamten Korpus generieren. Kontaktieren Sie uns.
  • Verarbeitungsgeschwindigkeit und Korpusgröße
    Sketch Engine wurde speziell für die schnelle Verarbeitung sehr großer Korpora entwickelt. Deswegen dauern Suchanfragen in Korpora mit weniger als einer Milliarde Wörter in der Regel nur wenige Sekunden. Bei Korpora mit mehr als einer Milliarde Wörter kann es ein wenig länger dauern. Wenn komplexe reguläre Ausdrücke (Regex) auf sehr große Korpora angewandt werden, kann die Verarbeitung mehrere Minuten dauern.
  • Voraussetzungen für die optimale Anwendung
    Ein %[token|tokenisiertes]% Korpus stellt die einzige Voraussetzung dar. Die Ergebnisse sind zwar repräsentativer, je größer das Korpus ist, es gibt jedoch keine erforderliche Mindestgröße für die Erstellung einer Wortliste.
